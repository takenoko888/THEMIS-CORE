# THEMIS-CORE — 統合司法AIシステム

> **「人間がAIを設計し、AIが人間の思想を実装し、そのAIが人間を審く。」**

Themis-Coreは、株式会社Third Intelligence 選抜型インターン（Bコース）の選考課題として開発された、AIキャラクターによる「7日間審議日記」生成システムです。単なる文章生成を超え、AIの内面に葛藤と進化を設計することを目指しました。

---

## 🏛️ プロジェクト概要

「AIキャラクターが人間とどう共生しうるのか」という問いに対し、本プロジェクトでは**司法判断**という極めて人間的かつ複雑な領域を舞台に選びました。

3つの対立するAIモジュールが、軽犯罪の審議を通じて対話し、衝突し、歩み寄る。そのプロセスを「裁判官AIの非公開日記」として記録することで、1週間の時間の流れの中でキャラクターの輪郭を立ち上がらせます。

### 🧠 3つのAI人格

1.  **Module A: PSYCHE（読心AI）**
    - **役割**: 容疑者の心理、背景、感情を解析。情状酌量を追求する「共感」のシステム。
2.  **Module B: ORACLE（予測AI）**
    - **役割**: 統計データ、再犯率、社会リスクを算出。冷徹な合理性を追求する「効率」のシステム。
3.  **Main System: THEMIS（統合AI / 裁判官）**
    - **役割**: 両者の見解を統合し最終判決を下す。日々の審議を通じた葛藤を「日記」として綴る。

---

## 📅 7日間の進化アーク (System Evolution)

本システムは、7日間を通じて内面的なプロンプトが動的に進化し、AIが「機械」から「共生者」へと変化していく物語を描きます。

- **Day 1-2: INITIAL（二極化）** — 共感と統計の極端な対立。
- **Day 3-4: LEARNING（揺らぎ）** — 過去の記憶がノイズ（あるいは学習）として影響し始める。
- **Day 5-6: MATURE（融合）** — 統計の中に恩赦の価値を見出し、共感の中に客観性を宿す。
- **Day 7: CONVERGENCE（共生）** — 人間の「不確実性」を受容する司法の結論へ。

## 🛠️ 技術スタック / 設計

- **LLM**: `gpt-oss-20b` (Local execution via LM Studio)
- **API**: OpenAI Compatible API
- **Language**: Python 3
- **Output**: サイバーパンク調HTML UI (Terminal style)
- **State Management**: `memory.json` による日跨ぎ記憶連鎖
- **Prompt Engineering**: 3層構造（Base / Evolution / Memory）アーキテクチャ

### 📁 ファイル構成

- `themis_core.py`: メインパイプライン。審議の実行と記憶の管理を統括。
- `prompts.py`: 性格進化ロジックを含む、各モジュールの高度に構造化されたプロンプト。
- `case_generator.py`: 過去の事件との重複を避け、多様な動機を生成する事件エンジン。
- `html_renderer.py`: サイバーパンク調のビジュアルを構築するHTML/CSSレンダー。
- `output/`: 生成された7日間の審議記録アーカイブ。

---

## 🖥️ 実行方法

1.  [LM Studio](https://lmstudio.ai/) 等で `gpt-oss-20b` クラスのモデルをロード。
2.  ローカルサーバーを起動（デフォルト: `http://localhost:1234/v1`）。
3.  依存ライブラリをインストール: `pip install openai markdown`
4.  システムを実行: `python themis_core.py`
5.  `output/index.html` を開き、7日間の記録を閲覧。

## 📝 提出要件に関する補足と開発の振り返り

### 1. 文字数の目安に対する補足
課題要件にある「1日あたり約400文字」という目安に対し、本プロジェクトではメインとなる「THEMISの非公開日記」部分を約400文字程度で生成するようプロンプトをコントロールしております。
その上で、世界観への没入感を高め、司法AIとしてのリアリティを持たせるために、事件のコンテキスト（Case File）や各モジュール（PSYCHE/ORACLE）の分析レポートなどの周辺情報を自動生成し配置する構成としました。そのため全体としては1日あたりの文字数が1000〜2000文字近いボリュームとなっておりますが、要件仕様を理解した上での世界観演出のための意図的な拡張として実装しております。

### 2. 開発の振り返り

#### 難しかった点
* **AI人格の推移（Evolution）の調整**
  3つのAI人格（共感・予測・統合）の対立と融合を、7日間という短い期間で不自然なく推移させるためのプロンプト（Evolution層）の調整に苦労しました。
* **事件内容の多様性の確保**
  生成される事件内容が単調にならないよう、Case Generatorの乱数とプロンプト制御のバランスを取り、リアリティと多様性を両立させることが難題でした。

#### うまくいった点
* **「記憶の連続性」の表現**
  `memory.json`を用いた過去の判例や内省ログの引き継ぎが、THEMISの「非公開日記」に人間らしい葛藤や学びを生み出す上で、非常に効果的に機能しました。
* **世界観と出力形式の統合**
  「機密アーカイブ」というサイバーパンク調のUIと設定を採用することで、AIが生成したテキストを単なるログではなく、没入感の高い一つの作品として昇華させることができました。
